Setup guide

Follow these steps in order. You only do this once. After setup, the form and tracker are fully live — future code updates never affect your data.

egf-submission-form.html
The form your team fills in to submit cases. Hosted on GitHub Pages.
egf-tracker.html
Your live case management dashboard. Reads and writes to Google Sheets.
egf-apps-script.gs
The backend. Lives in Google Sheets and handles all data operations.
1

Create a Google Sheet

This Sheet is your permanent database. You only create it once and never need to touch it again.

  1. Go to sheets.google.com
  2. Click + Blank spreadsheet
  3. Click "Untitled spreadsheet" at the top and rename it EGF Submissions
  4. Leave it empty — the script creates all columns automatically on first use
  5. Keep this tab open for the next step
Move this Sheet into a shared team Drive folder so Regan and forum members can access the raw data if needed.
2

Add the Apps Script

The Apps Script is the code that receives submissions and serves data to the tracker. It runs inside your Google Sheet.

  1. In your Google Sheet, click Extensions → Apps Script in the top menu bar
  2. A new browser tab opens with a code editor showing some default code
  3. Select all the default code and delete it (Ctrl+A then Delete)
  4. Open egf-apps-script.gs in a text editor on your computer
  5. Copy everything and paste it into the Apps Script editor
  6. Click the 💾 Save icon (or press Ctrl+S / Cmd+S)
  7. When prompted to name the project, type EGF Handler and click OK
Don't close this tab — you need it for the next step.
3

Deploy as a web app and copy the URL

Deploying turns your script into a URL that the form and tracker can talk to. This is the most important step.

  1. In the Apps Script editor, click the blue Deploy button (top right)
  2. Click New deployment
  3. Click the ⚙ gear icon next to "Select type" and choose Web app
  4. Fill in the settings exactly as shown below:
Required deployment settings
DescriptionEGF Handler v1
Execute asMe (your Google account)
Who has accessAnyone ← important
  1. Click Deploy
  2. Google will ask you to authorise the app — click "Authorise access"
  3. Sign in with your Xero Google account and click Allow
If Google shows a warning saying "This app isn't verified" — click Advanced then Go to EGF Handler (unsafe). This is normal for internal scripts.
  1. After authorising you'll see a screen with a Web app URL. It looks like this:
https://script.google.com/macros/s/AKfycbXXXXXXXXXXXXXXXXXXXX/exec
  1. Copy this URL — you need it in the next step. Keep it safe, don't share it publicly.
✓ You now have your Apps Script URL. This never changes unless you explicitly delete the deployment.
4

Add the URL to your GitHub files

You need to paste your Apps Script URL into two files on GitHub. Here's exactly how to do it for each file.

For egf-submission-form.html:

  1. Go to github.com/josieattard-bot/egf-governance
  2. Click on egf-submission-form.html
  3. Click the ✏ pencil icon (Edit this file)
  4. Press Ctrl+F (or Cmd+F on Mac) to open find — search for YOUR_APPS_SCRIPT_URL_HERE
  5. Replace it with your URL so the line looks exactly like this (keep the quotes):
const SCRIPT_URL = 'https://script.google.com/macros/s/AKfycb.../exec';
  1. Scroll down and click Commit changes → Commit changes again

For egf-tracker.html:

  1. Go back to the repo, click egf-tracker.html
  2. Click the ✏ pencil icon
  3. Search for YOUR_APPS_SCRIPT_URL_HERE and replace with your URL exactly as above
  4. Click Commit changes → Commit changes
✓ Wait about 60 seconds for GitHub to rebuild, then test your form at:
https://josieattard-bot.github.io/egf-governance/egf-submission-form.html
5

Set up AI assist (optional)

The AI assist feature uses the Claude API to help users draft their problem summary, risks, recommendation, and options. It's optional — the form works fully without it.

⚠ Your API key will be visible in the HTML file on GitHub. Only enable this if your GitHub repo is private, or if you are comfortable with the key being visible. Consider creating a restricted API key at console.anthropic.com.
  1. Go to console.anthropic.com and sign in (or create an account)
  2. Click API Keys → Create Key
  3. Name it EGF Form and copy the key (starts with sk-ant-…)
  4. In GitHub, edit egf-submission-form.html and find this line:
const ANTHROPIC_API_KEY = 'YOUR_ANTHROPIC_API_KEY_HERE';
  1. Replace YOUR_ANTHROPIC_API_KEY_HERE with your key and commit
Once the API key is added, the ✦ Draft with AI buttons activate on the Problem & Recommendation step. The AI uses everything the submitter has filled in (app name, segment, connections, JTBD, APIs accessed) to generate a contextual draft. Submitters review and edit before using it.

↻

Future code updates

When you get updated files from Claude:

  1. Go to the file on GitHub, click the ✏ pencil, select all and replace with the new code
  2. Search for YOUR_APPS_SCRIPT_URL_HERE and paste your URL back in
  3. If AI is set up, also replace YOUR_ANTHROPIC_API_KEY_HERE
  4. Commit — done. Your Google Sheets data is completely unaffected.
✓ This is the key improvement. The data lives in Google Sheets permanently — updating the HTML files never touches it.

Troubleshooting

Form says "Setup not complete"
The Apps Script URL hasn't been pasted in yet. Go to Step 4.

Tracker shows a red error bar
Check: (1) the URL in the tracker file is correct, (2) the deployment has "Who has access: Anyone", (3) you completed the authorisation in Step 3.

"This app isn't verified" Google warning
Click Advanced → Go to EGF Handler (unsafe). Normal for internal scripts.

No email notification received
Open Apps Script → Executions (left sidebar) to check for errors. Make sure Gmail permission was granted during authorisation.

Need to re-deploy after editing the Apps Script
Deploy → Manage deployments → click ✏ → set version to "New version" → Deploy. URL stays the same.

AI buttons don't appear / show error
Check the API key is pasted correctly with no extra spaces. Make sure it starts with sk-ant-. Check the browser console (F12) for specific error messages.

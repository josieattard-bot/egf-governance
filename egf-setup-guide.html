Setup guide

Connect your form and tracker to Google Sheets so all submissions are stored centrally — no data loss when files are updated, and accessible by anyone in your team.

0
How it works

You have three files. Once set up, they connect like this:

egf-submission-form.html
Submitters fill this in. On submit, data is sent to your Google Sheet automatically.
egf-tracker.html
You manage cases here. Reads live from the Sheet, updates status and notes back.
egf-apps-script.gs
The glue. A Google Apps Script that receives data from the form and serves it to the tracker.
The Apps Script acts as a secure API between your HTML files and Google Sheets. It runs on Google's servers — you don't need to host anything.
1
Create a Google Sheet

This Sheet will store all EGF submissions permanently. You only create it once.

  1. Go to sheets.google.com and click + Blank
  2. Name it "EGF Submissions" (top left, click "Untitled spreadsheet")
  3. Leave it empty — the Apps Script will create the header row and formatting automatically on the first submission
  4. Keep this tab open — you'll need it in Step 2
Tip: Move this Sheet into a shared team Drive folder so the whole governance team has access to the raw data as a backup.
2
Add the Apps Script

You'll paste the Apps Script code into Google's script editor, which is built into your Sheet.

  1. In your Google Sheet, click Extensions → Apps Script in the top menu
  2. A new tab opens showing a code editor with some default code. Delete everything in the editor
  3. Open the file egf-apps-script.gs in a text editor (Notepad, TextEdit, or VS Code)
  4. Copy the entire contents and paste it into the Apps Script editor
  5. If you want submissions emailed to a different address, find this line near the top and update it:
    const NOTIFY_EMAIL = 'josie.attard@xero.com';
  6. Click the 💾 Save icon (or Cmd/Ctrl + S). Name the project "EGF Handler" when prompted
Don't close this tab yet — you need to deploy it in the next step.
3
Deploy as a web app

Deploying turns your script into a URL that the form and tracker can talk to. This is the most important step.

  1. In the Apps Script editor, click Deploy → New deployment (top right button)
  2. Click the gear icon ⚙ next to "Select type" and choose Web app
  3. Fill in the deployment settings:
    Required settings
    Description: EGF Submission Handler v1
    Execute as: Me (your Google account)
    Who has access: Anyone
  4. Click Deploy
  5. Google will ask you to authorise the app — click "Authorise access", sign in with your Xero Google account, and click "Allow". This lets the script read/write your Sheet and send emails on your behalf.
  6. After authorising, you'll see a screen with a Web app URL. It looks like:
    https://script.google.com/macros/s/AKfycb.../exec
  7. Copy this URL — you'll need it in the next step
⚠ Keep this URL private — anyone with it can submit to your Sheet. Don't post it publicly.
If Google shows a warning saying "This app isn't verified" — click Advanced → Go to EGF Handler (unsafe). This is normal for internal scripts that haven't gone through Google's verification process.
4
Paste the URL into both HTML files

Open both HTML files in a text editor and replace the placeholder with your real Apps Script URL.

  1. Open egf-submission-form.html in a text editor
  2. Find this line near the bottom of the file:
    const SCRIPT_URL = 'YOUR_APPS_SCRIPT_URL_HERE';
  3. Replace YOUR_APPS_SCRIPT_URL_HERE with your URL (keep the quotes):
    const SCRIPT_URL = 'https://script.google.com/macros/s/AKfycb.../exec';
  4. Save the file
  5. Repeat steps 1–4 for egf-tracker.html — it has the same placeholder line near the bottom
✓ Once both files have the URL, the setup is complete. Test it by opening the form, filling it in, and submitting — then open the tracker and click Refresh to see the submission appear.
5
Share with your team

Now that the files are connected to Sheets, anyone can use them — they just need access to the files.

  1. Upload both HTML files (egf-submission-form.html and egf-tracker.html) to a shared Google Drive folder
  2. For the submission form: share the link with anyone who needs to submit cases (DE managers, regional leads, etc.)
  3. For the tracker: share with the governance team only (Josie, Regan, and forum members)
  4. To open an HTML file from Google Drive: right-click → Open with → Google Chrome
Alternatively, embed either file in a Confluence page using the HTML macro (Insert → Other Macros → HTML). This gives a permanent internal URL that anyone can bookmark — no Drive required.
6
Future code updates

This is the key improvement over the old local storage approach.

  1. When you get updated HTML files, open them in a text editor and find the SCRIPT_URL line
  2. Paste in your Apps Script URL (same one as before)
  3. Upload the new files to Drive, replacing the old ones
  4. All your submissions in Google Sheets are completely unaffected — the Sheet is separate from the HTML files
✓ If the Apps Script itself needs updating, deploy a new version in Apps Script (Deploy → Manage deployments → Edit → New version) — the URL stays the same.
7
Troubleshooting

Common issues and fixes:

Form says "Setup not complete"
You haven't pasted the Apps Script URL into the form file yet. See Step 4.

Tracker shows "Could not connect to Google Sheets"
Check that: (1) the URL in the tracker file matches your deployed URL exactly, (2) the deployment is set to "Anyone" for access, (3) you authorised the script in Step 3.

Google shows "This app isn't verified" warning
This is expected for internal scripts. Click Advanced → Go to EGF Handler (unsafe) to proceed.

Submissions aren't appearing in the Sheet
Open the Apps Script editor and click Run → doGet to test it manually. Check the execution log (View → Executions) for errors.

No email notification on submission
The script sends email using your Google account. Make sure you granted Gmail permissions during authorisation. Check your Sent folder — if nothing is there, re-deploy the script and re-authorise.

Need to re-deploy after changes
Go to Apps Script → Deploy → Manage deployments → click the pencil ✏ → change version to "New version" → Deploy. The URL doesn't change.
